# 品証パニック — 2人オンライン対戦 実装指示書

## 概要

ソロモード（Phase 1-3）のコアメカニクスをベースに、Firebase Realtime Databaseを使った2人オンライン対戦モードを実装する。

- **技術スタック**: 既存のReact + Vite + TypeScript
- **バックエンド**: Firebase Realtime Database
- **認証**: 匿名認証（Firebase Anonymous Auth）
- **マッチング**: ルームコード方式（4桁コード）

---

## ゲームルール（2人対戦）

### ソロとの差分

| 項目 | ソロ | 2人対戦 |
|---|---|---|
| ラウンド数 | 8R | 7R |
| 山札 | 1人で使う | 共有（交互にめくる） |
| 汚染投入 | 自分の利益÷2 | 両者の利益合計÷2 |
| 汚染キャップ | 3枚/R | 3枚/R |
| パニック汚染 | 3枚 | 3枚（相手も巻き込まれる） |
| 勝利条件 | 目標スコア到達 | 7R終了時にスコアが高い方の勝ち |
| 対応カード初期手札 | 1枚 | 1枚 |
| 同点時 | — | 対応カード残りが少ない方の勝ち |

### ラウンドの流れ（2人）

```
[準備フェーズ] ※ホスト側で処理、結果を同期
  R1: 汚染なし
  R2以降:
    投入枚数 = ceil(両者の前ラウンド確定利益合計 / 2)
    キャップ: 最大3枚/R
    出荷山札をシャッフル

[出荷フェーズ] ※手番プレイヤーが操作、相手は観戦
  手番順: P1 → P2 → ラウンド終了
  各プレイヤーの手番:
    - ソロと同じめくりルール
    - 相手は手番の様子をリアルタイムで観戦
    - パニック発生 → 汚染3枚追加（次の手番プレイヤーに影響）

[結果処理]
  両者の手番終了後:
    - 利益確定、対応カード入手処理
    - 汚染投入（両者の合計利益ベース）
    - 次ラウンドへ

[ゲーム終了]
  7R終了 or 山札切れ → スコア比較 → 勝敗判定
```

## Firebase データ構造

```typescript
// Realtime Database のルーム構造
interface Room {
  // ルーム情報
  roomCode: string;          // 4桁ルームコード
  status: 'waiting' | 'playing' | 'finished';
  createdAt: number;         // タイムスタンプ
  
  // プレイヤー
  players: {
    [uid: string]: {
      name: string;
      ready: boolean;
      score: number;
      responseHand: ResponseCard[];  // 対応カード手札
      connected: boolean;
    }
  };
  hostUid: string;           // ホストのUID
  playerOrder: string[];     // [hostUid, guestUid] 手番順
  
  // ゲーム状態（ホストが管理）
  gameState: {
    phase: 'waiting' | 'prepare' | 'shipping' | 'result' | 'game_over';
    round: number;
    maxRounds: number;       // 7
    
    // 山札（ホストが管理、カードIDの配列）
    drawPile: string[];      // カードIDの配列
    contaminationStock: string[];
    
    // 対応カード（共有ストック）
    responseStock: string[];
    responseDiscard: string[];
    
    // 現在の手番
    currentPlayerUid: string;
    currentPlayerIndex: number;  // 0 or 1
    
    // 手番中の状態
    turnState: {
      drawnCards: string[];      // めくったカードID
      currentProfit: number;
      currentDefectPoints: number;
      panicThreshold: number;
      snsFireActive: boolean;
      forcedDraws: number;
      waterInspectionActive: boolean;
      isPanicked: boolean;
    } | null;
    
    // ラウンド結果
    roundResults: {
      [round: number]: {
        [uid: string]: {
          profit: number;
          panicked: boolean;
          cardsDrawn: number;
        }
      }
    };
    
    // 汚染情報（演出用）
    lastContamination: {
      count: number;
      round: number;
    } | null;
  };
  
  // カードマスタデータ（ルーム作成時にホストが生成）
  cardMaster: {
    [cardId: string]: Card;
  };
}
```

## 画面遷移

```
[タイトル画面]
  ├── ソロプレイ → 既存のソロモード
  └── 2人対戦
       ├── ルームを作る → ルームコード表示 → 待機画面
       └── ルームに入る → コード入力 → 待機画面

[待機画面]
  - 2人揃ったら「ゲーム開始」ボタン（ホストのみ）
  - 相手の接続状態表示

[ゲーム画面]
  - 自分の手番: めくる/止めるの操作可能
  - 相手の手番: 観戦モード（相手のめくりがリアルタイムで見える）
  - 共通: 山札残数、両者のスコア、ラウンド情報

[結果画面]
  - 勝敗表示
  - 両者のラウンドごとのスコア比較
  - もう一度遊ぶ / タイトルに戻る
```

## 実装の優先順位

### Step 1: Firebase セットアップ
1. Firebase プロジェクト作成（既存のがあればそれを使う）
2. Realtime Database のルール設定
3. 匿名認証の有効化
4. firebase.ts に設定ファイル作成

### Step 2: ルーム管理
1. ルーム作成（4桁コード生成、ホスト登録）
2. ルーム参加（コード入力、ゲスト登録）
3. 待機画面（接続状態の監視）
4. 切断検知（onDisconnect）

### Step 3: ゲーム同期
1. ホスト側でゲーム初期化（山札生成、カードマスタ登録）
2. 手番管理（currentPlayerUidで制御）
3. めくりアクションの同期（ホストがめくり→結果をDB書き込み→両者に反映）
4. 対応カード使用の同期

### Step 4: 観戦・UI
1. 相手の手番中のリアルタイム表示
2. 両者のスコアボード
3. パニック・イベントの相手側への通知
4. ゲーム終了・勝敗判定

### Step 5: エッジケース
1. 切断時の処理（タイムアウトで相手の勝ち）
2. ルームの自動削除（24時間後）
3. 不正操作防止（ホスト側でのみ山札操作）

## ゲームロジックの役割分担

```
ホスト（P1）の責務:
  - 山札の生成・シャッフル
  - 汚染投入の計算・実行
  - めくり結果の判定（自分・相手とも）
  - ラウンド進行の管理

ゲスト（P2）の責務:
  - 自分の手番で「めくる」「止める」の入力のみ
  - 対応カードの使用選択
  - DBの変更をリッスンして画面更新

共通:
  - DBのリアルタイムリスナーで状態同期
  - カードマスタからIDでカード情報を解決
```

---

## 重要な設計方針

1. **ホスト権限方式**: ゲームロジックの判定はホスト側のみで行う。ゲストは入力（めくる/止める/対応カード使用）をDBに書き込み、ホストがそれを検知して処理結果を書き戻す。これにより不正防止とデータ整合性を担保する。

2. **カードIDシステム**: カード実体はcardMasterに一度だけ書き込み、以降はIDで参照。通信量を最小化する。

3. **既存のgameEngine.tsを再利用**: ソロモードのゲームロジック（純粋関数）をそのまま2人対戦でも使う。Firebase層はgameEngineのラッパーとして機能する。

4. **楽観的UI更新はしない**: 全てDBの変更をリッスンして画面更新。遅延は許容する（リアルタイムDBなのでほぼ即座）。

## Firebase Realtime Database ルール

```json
{
  "rules": {
    "rooms": {
      "$roomCode": {
        ".read": "auth != null",
        ".write": "auth != null",
        "gameState": {
          ".write": "data.parent().child('hostUid').val() === auth.uid || !data.exists()"
        }
      }
    }
  }
}
```

---

## 注意事項

- Firebase の設定値（apiKey等）は環境変数（.env）で管理する
- ルームコードは4桁英数字、既存コードとの重複チェックを行う
- ゲーム中に片方が切断した場合、30秒のタイムアウト後に相手の勝ちとする
- 既存のソロモードは影響を受けないようにする（タイトル画面からの分岐）
